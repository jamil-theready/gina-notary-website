<objective>
Remove next-sitemap completely and audit sitemap.ts to ensure noindex pages are excluded.

Purpose: The build currently runs `next build && next-sitemap`, which generates a competing `out/sitemap.xml` that overwrites the App Router sitemap. Search engines receive the wrong sitemap. This fixes CRIT-02 (single sitemap system) and CRIT-03 (noindex pages excluded). The App Router `sitemap.ts` already excludes `/thank-you/` and `/notary-resources/` from its static pages array, but the file needs verification and a redirect guard to ensure `/notary-resources/` (which redirects) never leaks back in.

Output: next-sitemap uninstalled, config file deleted, build script trimmed to `next build`, sitemap.ts verified clean.
</objective>
